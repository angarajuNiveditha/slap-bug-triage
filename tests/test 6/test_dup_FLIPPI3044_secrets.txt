Subject: Edison service failing to fetch secrets — Grayskull not integrated

Hi Team,

The Edison service is unable to read secrets properly. We have not yet
integrated Grayskull for secrets management in Edison, so it is falling
back to an older mechanism that is unreliable in production.

This is causing intermittent failures when Edison tries to access
credentials at runtime. The service starts fine but fails silently
when it needs to resolve a secret value.

Steps to Reproduce:
1. Deploy Edison service to production environment
2. Trigger any flow that requires secret resolution (API keys, DB creds)
3. Edison fails to fetch the secret via the current mechanism
4. Service returns an error or uses a stale/empty value

Expected: Edison fetches secrets securely via Grayskull integration.
Actual: Edison is not using Grayskull at all — secrets fetch fails
        intermittently using the legacy approach.

Reproducibility: Intermittent in prod, consistent when Grayskull
                 is unavailable.

Impact:
Any Edison-dependent flow that relies on secrets is at risk. This is
a security and reliability concern. Needs Grayskull integration to
be properly set up in Edison before next release.

Environment:
- Service: Edison
- Secrets manager: Grayskull (not yet integrated)
- Environment: Production
