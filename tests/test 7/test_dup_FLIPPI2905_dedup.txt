Subject: Duplicate products showing up in journey continuation search results

Hi,

There is an issue with the search results returned during journey
continuation. When a user resumes a previous shopping session and
searches for a product, the same product is appearing multiple times
in the results — different variants of the same product family are
shown as separate items instead of being grouped.

This is cluttering the search results and making it hard for users
to compare options cleanly.

Steps to Reproduce:
1. Start a shopping session in SLAP and search for a product
2. Exit the session and re-enter (journey continuation)
3. Perform the same or similar search
4. Observe the search results

Expected: Products from the same family should be deduplicated or
          grouped — only the best/most relevant variant shown.
Actual: Multiple variants of the same product family appear as
        separate cards, causing duplicate-looking results.

Reproducibility: Consistent — happens on every journey continuation
                 search.

Impact:
Search result quality is poor for returning users. Increases
cognitive load and may lead to drop-offs. Affects all platforms.

Environment:
- Platform: Android, iOS, Web (all affected)
- Feature: Journey continuation search
- App Version: 2.4.1
